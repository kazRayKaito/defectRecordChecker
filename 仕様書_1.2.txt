--------------------------------------------------------------------------------
1. システム概要
--------------------------------------------------------------------------------
名称    : defectRecordChecker
目的    : 製造ラインにおける不良品検査記録の確認・出力を行うデスクトップアプリ
対象OS  : Windows 10 / 11
言語    : Python 3.x
主要ライブラリ: tkinter, Pillow (PIL)

--------------------------------------------------------------------------------
2. 画面構成
--------------------------------------------------------------------------------
ウィンドウサイズ: 1000 x 700 px

【全体レイアウト】
  ┌────────────────┬──────────────────────────────────────────┐
  │  左パネル       │  右パネル                                 │
  │  工程進捗表示   │  フォルダパス入力 / 画像表示 / ナビゲーション│
  │  （固定幅260px）│  （可変幅）                                │
  └────────────────┴──────────────────────────────────────────┘

【左パネル：工程進捗（縦一列ツリー構造）】

  工程進捗
  ────────────────────────
  STEP 1  ケースQR読取
    ● 読取OK
    ● 流動情報への問い合わせOK
    ● 流動情報の内容OK
  ────────────────────────
  STEP 2  NG画像確認
    ● NG画像の取得OK
    ● X/N枚目完了          ← 動的更新。全枚確認で緑点灯
  ────────────────────────
  STEP 3  全画像確認
    ● 全画像の取得OK
    ● X/N枚目完了          ← 動的更新。全枚確認で緑点灯
  ────────────────────────
  STEP 4  結果出力
    [書き込み実施]          ← STEP1〜3全完了時のみ押下可能
    ● CSV記録書き込みOK     ← ボタン押下後に順次点灯
    ● F-IoT書き込みOK
  ────────────────────────

  ※ 各インジケーター（●）の色仕様:
      グレー (#9CA3AF) = 未完了
      青    (#3B82F6) = 進行中（X/N枚目完了 の進捗中）
      緑    (#22C55E) = 完了

【右パネル】
  - フォルダパスの入力テキストボックス
  - NG画像読込ボタン / 全画像読込ボタン
  - ファイル名・枚数ラベル（例: [3/10]  filename.jpg）
  - 画像表示エリア（最大 500x500px、アスペクト比維持）
  - ナビゲーションボタン: ◀前へ / 次へ▶ / ✓確認完了

--------------------------------------------------------------------------------
3. 機能仕様
--------------------------------------------------------------------------------

【3-1. STEP1：ケースQR読取】（未実装 / 条件別途決定）

  サブステップと完了条件:
    (a) 読取OK              … QRコード読取成功（完了条件: 別途）
    (b) 流動情報への問い合わせOK … 流動情報システムへの問い合わせ成功（完了条件: 別途）
    (c) 流動情報の内容OK     … 取得した流動情報の内容が正しい（完了条件: 別途）

  STEP1完了条件: (a)(b)(c) すべて緑

【3-2. STEP2：NG画像確認】（一部実装済み）

  サブステップと完了条件:
    (a) NG画像の取得OK       … 指定フォルダからJPGファイルを1枚以上取得できた場合に緑
    (b) X/N枚目完了          … 以下の動作に従い更新:
          - 「次へ▶」クリック時: X がインクリメント（前の画像が確認済み扱い）
          - 最後の1枚で「✓確認完了」クリック時: X = N となり緑点灯
          - ラベルは常に「X/N枚目完了」の形式で単一行表示（N行に展開しない）

  STEP2完了条件: (a)(b) ともに緑（ng_done >= ng_total）

【3-3. STEP3：全画像確認】（未実装 / 13枚程度を想定）

  サブステップと完了条件:
    (a) 全画像の取得OK       … 指定フォルダからJPGファイルを1枚以上取得できた場合に緑
    (b) X/N枚目完了          … STEP2と同様の動作

  STEP3完了条件: (a)(b) ともに緑（all_done >= all_total）

【3-4. STEP4：結果出力】（未実装）

  前提条件: STEP1〜3 がすべて完了している場合のみ「書き込み実施」ボタンが押下可能。

  サブステップと完了条件:
    (a) CSV記録書き込みOK   … 書き込み実施ボタン押下後に書き込み成功 → 緑点灯
    (b) F-IoT書き込みOK     … CSV書き込み完了後に連続して書き込み成功 → 緑点灯

  処理順序: ボタン押下 → CSVに書き込み → (a)点灯 → F-IoTに書き込み → (b)点灯

--------------------------------------------------------------------------------
4. 画像ナビゲーション仕様
--------------------------------------------------------------------------------

  - 「NG画像読込」ボタン: 入力パスからJPGを取得し、STEP2用画像セットとして登録
  - 「全画像読込」ボタン: 入力パスからJPGを取得し、STEP3用画像セットとして登録
  - どちらを読み込んだかによって現在のモード（ng / all）が切り替わる

  ナビゲーション:
    ◀前へ : 1枚前の画像を表示（最初の1枚では無効）
    次へ▶ : 1枚次の画像を表示し、前の画像を確認済みカウントに加算（最後の1枚では無効）
    ✓確認完了: 最後の1枚を表示中かつ未完了の場合のみ有効。押下でステップ完了

  完了カウント:
    done = 「次へ▶」で通過した枚数。最後で「✓確認完了」押下時に done = total。
    戻った場合でも done は減少しない（最大値を保持）。

--------------------------------------------------------------------------------
5. 入出力仕様
--------------------------------------------------------------------------------

  【入力】
    - フォルダパス（テキスト入力）
    - 対象ファイル形式: *.jpg（ファイル名昇順ソート）

  【出力】
    - 画面への画像表示（最大 500x500 px、アスペクト比維持）
    - CSV記録ファイル（書き込み仕様は別途決定）
    - F-IoTへの書き込み（仕様は別途決定）

--------------------------------------------------------------------------------
6. 定数・設定値
--------------------------------------------------------------------------------

  定数名        値          説明
  ------------- ----------- ---------------------------
  C_DONE        #22C55E     完了インジケーター色（緑）
  C_ACTIVE      #3B82F6     進行中インジケーター色（青）
  C_PENDING     #9CA3AF     未完了インジケーター色（グレー）
  C_PANEL       #F3F4F6     左パネル背景色
  左パネル幅     260 px      固定
  画像最大サイズ  500x500 px  アスペクト比維持
